# News（投稿）本番 → dev 移行手順

本番の News（投稿）を dev へ移行する際の具体的な手順です。

---

## 前提

- **News** = WordPress 標準の「投稿」（post_type: post）
- **本番**: https://nubow.co.jp/news
- **dev**: https://dev.nubow.co.jp/news
- **URL 形式**: `/news/カテゴリスラッグ/記事スラッグ`（例: `/news/取材・メディア情報/記事タイトル`）

---

## Phase 1: 事前確認（移行前に実施）

### 1-1. News カテゴリの確認

| 本番で確認 | dev で確認 |
|-----------|------------|
| 管理画面 → 投稿 → カテゴリ | 同様にカテゴリ一覧を確認 |
| お知らせ / 取材・メディア情報 / 店舗情報 など | slug・名前が一致しているか |

**重要**: dev にカテゴリがない場合、本番と同じ slug・名前で作成しておく。

### 1-2. 固定ページ「news」の確認

- dev に固定ページ「news」（スラッグ: news）が存在するか確認
- テンプレート「お知らせ一覧」が割り当てられているか確認

---

## Phase 2: エクスポート（本番）

### 重要: 投稿だけをエクスポートする方法

**All-in-One WP Migration の無料版では「投稿のみ」のエクスポートはできません。**  
サイト全体がエクスポートされます。投稿だけ移行したい場合は、下記の **WordPress 標準エクスポート** を使います。

### 方法A: WordPress 標準エクスポート（投稿のみ）※推奨

1. **本番** (https://nubow.co.jp/wp-admin) にログイン
2. 左メニュー **「ツール」** → **「エクスポート」**
3. 画面のラジオボタンで **「投稿」** を選択
4. **「エクスポートファイルをダウンロード」** をクリック
5. XML ファイル（例: `nubow.wordpress.2025-xx-xx.xml`）がダウンロードされる

- 投稿の本文・カテゴリ・タグ・著者・日付などが含まれる
- メディア（画像）は URL 参照のみ。実ファイルは含まれない
- 画像も必要なら「添付ファイルをダウンロードしてインポート」で取り込む（Phase 3 参照）

---

## Phase 3: インポート（dev）

1. **dev** (https://dev.nubow.co.jp/wp-admin) にログイン（Basic認証: admin / nubow）
2. 左メニュー **「ツール」** → **「インポート」**
3. **「WordPress」** の「今すぐインストール」をクリック  
   （すでにインストール済みなら「インポーターを実行」）
4. ダウンロードした **XML ファイル** を選択してアップロード
5. **「投稿者をマッピング」**:  
   本番の著者名に対して、dev のユーザー（例: admin）を選択して「このユーザーをインポート先とする」で確定
6. **「添付ファイルをダウンロードしてインポート」** にチェックを入れる  
   （アイキャッチ・本文内画像も取り込む場合。チェックしないとテキストのみ）
7. **「インポートを実行」** をクリック

---

## Phase 4: インポート後の URL 置換

インポートした記事の本文・メタデータなどに本番 URL が残っているため、dev 用に置換します。

### Better Search Replace を使う場合（推奨）

1. dev に [Better Search Replace](https://ja.wordpress.org/plugins/better-search-replace/) をインストール・有効化
2. ツール → Better Search Replace を開く
3. **以下の 2 セットを、順番に実行**：

---

#### 置換 1

| 項目 | 入力する内容 |
|------|--------------|
| **Search for（検索）** | `https://nubow.co.jp` |
| **Replace with（置換）** | `https://dev.nubow.co.jp` |

- 「Select tables」: **「Run on entire database」** にチェック（全テーブル対象）
- 「Run as dry run?」: **チェックを外す**（本番実行）
- 「Submit」をクリック

---

#### 置換 2

| 項目 | 入力する内容 |
|------|--------------|
| **Search for（検索）** | `http://nubow.co.jp` |
| **Replace with（置換）** | `https://dev.nubow.co.jp` |

- 同上
- 「Submit」をクリック

---

**補足**:
- 「Run as dry run」にチェックを入れると、実際には置換せず結果だけ表示される（まず確認したいときに使う）
- データベースのバックアップを取ってから実行することを推奨

## Phase 5: 移行後チェック

- [ ] https://dev.nubow.co.jp/news で記事一覧が表示される
- [ ] カテゴリが正しく紐づいている
- [ ] ページネーション（/news/page/2 など）が動く
- [ ] 個別記事リンクが開ける
- [ ] アイキャッチ・本文内画像が表示される（メディアを移行した場合）

## トラブルシュート

| 現象 | 対処 |
|------|------|
| 画像が表示されない | メディアを別途移行するか、uploads を FTP でコピー |
| カテゴリが紐づかない | インポート前に dev でカテゴリ（slug 一致）を作成 |
| ページネーション 404 | 設定 → パーマリンク を開いて「変更を保存」でリライトルール再生成 |
| 記事が表示されない | 固定ページ「news」が存在し、表示設定で「投稿ページ」が正しく設定されているか確認 |
| `/uncategorized/766` など変なURLになる | 下記「変なURLになる場合」を参照 |

---

### 変なURLになる場合（/uncategorized/記事ID など）

**原因**:
1. **表示設定**で「投稿ページ」が「news」になっていない
2. インポート後、記事のカテゴリが「未分類」のままになっている

**対処**:

#### 1. 表示設定の確認

1. dev 管理画面 → **設定** → **表示設定**
2. **「投稿ページ」** で **「news」（お知らせ）** を選択
3. **「変更を保存」** をクリック

これで投稿のベースURLが `/news` になります。

#### 2. カテゴリの再割り当て

1. **投稿** → **一覧**
2. `/uncategorized/` になっている記事を複数選択
3. **一括操作** で「編集」を選択 → **適用**
4. **カテゴリ** で正しいカテゴリ（お知らせ / 取材・メディア情報 / 店舗情報 など）にチェック
5. **一括更新** をクリック

#### 3. パーマリンクの再生成

1. **設定** → **パーマリンク**
2. 何も変えずに **「変更を保存」** をクリック

リライトルールが再生成され、正しいURLで表示されるようになります。

#### 4. テーマの functions.php によるURL形式の強制（テーマ側）

テーマの `functions.php` には、投稿URLを `/news/カテゴリスラッグ/記事スラッグ` 形式に強制するコードが入っています。  
カテゴリは紐づいているのに `/uncategorized/記事ID` になってしまう場合は、以下を確認してください。

- テーマが最新版（`functions.php` にリライトルールと `post_link` フィルターが含まれているか）
- **設定 → パーマリンク** で一度「変更を保存」をクリックし、リライトルールをフラッシュ

これで `get_permalink()` や `the_permalink()` が `/news/カテゴリ/記事` 形式を返すようになります。
